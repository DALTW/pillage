# 문서 목적

Pencil Village는 일반적인 대사 텍스트와 튜토리얼 문장을 최소화한다. 이 문서는 플레이어가 텍스트 없이도 무엇을 해야 하는지 이해할 수 있도록 만드는 상호작용 UX 규칙을 정의한다.

# UX 원칙

## 1. 말보다 그림

NPC의 요청, 보상, 목표, 상호작용 가능 여부는 아이콘과 시각 효과로 전달한다.

## 2. 막히지 않는 유도

플레이어에게 노골적인 화살표를 계속 보여주지 않는다. 대신 반짝임, 느낌표, 카메라 시선, 오브젝트 위치 배치로 자연스럽게 유도한다.

## 3. 일관된 상호작용 문법

상호작용 가능한 대상은 모두 같은 규칙을 따른다. 플레이어가 한 번 규칙을 이해하면 NPC, 아이템, 스케치북, 문에 모두 적용할 수 있어야 한다.

# 텍스트 사용 규칙

MVP 기준으로 다음 텍스트는 사용하지 않는다.

- NPC 대사 문장
- 퀘스트 설명 문장
- 튜토리얼 설명 문장
- 선택지 텍스트
- 목표 문장

다음은 허용 가능하다.

- 키캡 안의 상호작용 키 표시
- 개발 중 디버그용 텍스트
- 발표용 외부 설명 자료

게임 안에서는 키캡 모양 이미지로 상호작용 키를 표시한다.

# 상호작용 힌트 규칙

상호작용 가능한 대상은 다음 단계로 표현한다.

```text
멀리 있음 -> 대상이 은은하게 반짝임
가까이 접근 -> 플레이어 머리 위 말풍선 표시
말풍선 안 -> 상호작용 키캡 표시
키 입력 -> 상호작용 실행
```

## 반짝임

반짝임은 상호작용 가능한 오브젝트를 찾게 만드는 기본 힌트다.

적용 대상:

- NPC
- 연필 조각
- 크레파스
- 퀘스트 오브젝트
- 스케치북
- 최종 문

반짝임은 과하지 않아야 한다. 힐링 게임 분위기를 해치지 않도록 은은하고 부드럽게 표현한다.

## 플레이어 머리 위 말풍선

플레이어가 상호작용 거리 안으로 들어오면, 오브젝트 위가 아니라 플레이어 머리 위에 말풍선이 뜬다.

장점:

- 화면에 UI가 난잡하게 쌓이지 않는다.
- 플레이어가 현재 상호작용할 수 있음을 즉시 인식한다.
- 모든 오브젝트에 같은 UX를 적용할 수 있다.

## 키캡 아이콘

말풍선 안에는 키캡 모양 이미지가 표시된다. 키캡 안에 실제 키 문자가 들어가도 된다.

권장 예시:

```text
[ E ]
```

문자 없는 게임이지만, 조작 키 표시로서의 `E`는 허용한다. 해커톤 시연에서 조작을 몰라 진행이 막히는 리스크를 줄이기 위해서다.

# 진행 대상 느낌표

연필이 완성되면 오두막 위에 느낌표가 나타난다. 오두막 내부로 들어가면 스케치북 위에도 느낌표가 나타난다.

이 느낌표는 일반 상호작용 힌트가 아니라 **현재 중요한 진행 이벤트가 있음**을 나타낸다.

사용 조건:

- 연필이 완성되어 숲을 확장할 수 있을 때
- 스케치북에 적용할 수 있는 연필 또는 크레파스가 있을 때
- 마지막 문이 등장했을 때 필요하면 사용 가능

# 자연스러운 유도 방식

## 오두막 복귀 유도

연필이 완성되면 다음 흐름을 사용한다.

```text
연필 완성
-> 오두막 위 느낌표 표시
-> 플레이어가 오두막 근처 접근
-> 아이의 시선 또는 카메라가 오두막 쪽을 살짝 유도
-> 플레이어가 오두막으로 들어감
-> 스케치북 위 느낌표 표시
```

화살표를 직접 띄우지 않고도 진행 방향을 알 수 있다.

## 스케치북 목표 유도

오두막 내부에서는 스케치북 위에 느낌표를 표시한다. 플레이어가 스케치북 근처에 가면 머리 위 말풍선과 키캡이 표시된다.

# NPC 말풍선 문법

NPC는 텍스트 없이 아이콘 말풍선으로 퀘스트를 전달한다.

## 기본 구조

```text
요청: [필요한 물건 아이콘] + [?]
완료: [필요한 물건 아이콘] + [하트]
보상: [연필 조각] 또는 [크레파스]
```

## 예시

우체통 주민:

```text
[편지] + [?]
```

작은 새:

```text
[나뭇가지] + [?]
```

다람쥐:

```text
[도토리] + [?]
```

큰 나무:

```text
[빛나는 잎] + [?]
```

# 보상 UX

퀘스트 완료 후 보상은 자동으로 인벤토리에 들어가지 않는다.

흐름:

```text
퀘스트 완료
-> NPC가 기뻐하는 아이콘 표시
-> 보상 오브젝트가 필드에 떨어짐
-> 보상이 반짝임
-> 플레이어가 가까이 감
-> 머리 위 키캡 말풍선 표시
-> 상호작용으로 획득
```

이 방식은 보상을 직접 얻는 감각을 강화한다.

# 상호작용 대상별 처리

## NPC

- 가까이 가면 말풍선과 키캡 표시
- 상호작용 시 요청 아이콘 표시
- 필요한 오브젝트를 보유하면 완료 상태로 전환

## 퀘스트 오브젝트

- 반짝임
- 가까이 가면 키캡 말풍선
- 상호작용하면 획득 상태로 저장

## 연필 조각

- 반짝이는 작은 연필 조각으로 표시
- 상호작용 시 연필 UI에 조각 추가
- 3개가 되면 연필 완성 연출

## 크레파스

- 보상으로 필드에 떨어짐
- 상호작용 시 보유 크레파스 목록에 추가
- 필드 색은 즉시 바뀌지 않음

## 스케치북

- 항상 상호작용 가능
- 업그레이드 물건이 없으면 지도 화면만 표시
- 연필 또는 크레파스가 있으면 한 번에 하나씩 적용

## 최종 문

- 필드 숲 끝에 등장
- 오두막 문과 닮은 형태
- 가까이 가면 키캡 말풍선
- 상호작용하면 엔딩 시작

# 목표 UI

목표 UI는 최소화한다. 확정된 목표 UI는 다음 하나다.

```text
완성된 연필 아이콘 + 오두막 아이콘
```

이는 연필이 완성되어 오두막으로 돌아가야 함을 의미한다.

나머지 목표는 NPC 말풍선, 반짝임, 스케치북 변화, 필드 배치로 전달한다.

# UX 리스크와 대응

## 리스크 1: 플레이어가 퀘스트 목표를 이해하지 못함

대응:

- 아이콘은 매우 구체적인 물건으로 사용한다.
- 편지, 나뭇가지, 도토리, 빛나는 잎처럼 형태가 명확한 물건을 사용한다.
- 퀘스트 오브젝트는 NPC 주변 또는 새로 해금된 구역 안에 배치한다.

## 리스크 2: 상호작용 키를 모름

대응:

- 상호작용 거리 안에 들어오면 플레이어 머리 위에 키캡을 표시한다.
- 키캡은 실제 키와 손 아이콘을 함께 보여준다.

## 리스크 3: 오두막으로 돌아가야 하는지 모름

대응:

- 연필 완성 시 오두막 위에 느낌표를 표시한다.
- 오두막 근처 접근 시 카메라 또는 시선 유도를 사용한다.

# UX 완성 기준

Interaction UX가 제대로 구현되었다고 판단하려면 다음이 가능해야 한다.

- 플레이어가 텍스트 없이 NPC의 요청을 대략 이해한다.
- 반짝이는 오브젝트를 상호작용 대상으로 인식한다.
- 키캡 말풍선을 보고 상호작용 키를 누른다.
- 연필 완성 후 오두막으로 돌아가야 함을 이해한다.
- 스케치북에서 세계가 확장된다는 규칙을 이해한다.
