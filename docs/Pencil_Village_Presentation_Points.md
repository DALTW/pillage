# 문서 목적

이 문서는 Pencil Village를 해커톤에서 발표하거나 시연할 때 강조할 내용을 정리한다. 짧은 시간 안에 게임의 의도, 차별점, 구현 방식, AI 코딩 활용 포인트를 전달하는 것이 목적이다.

# 한 줄 소개

**Pencil Village는 아이가 그린 스케치북 세계를 탐험하고, 연필 조각과 크레파스로 세계를 완성해 현실로 돌아가는 힐링 브라우저 게임입니다.**

# 30초 소개

Pencil Village는 통나무 오두막에서 스케치북을 그리던 아이가 자신이 그린 마을과 숲 속으로 들어가게 되는 게임입니다. 플레이어는 NPC를 돕고 숲을 탐험하며 연필 조각을 모읍니다. 연필이 완성되면 오두막 안의 스케치북에 새로운 숲을 그리고, 그렇게 세계를 오른쪽 영역으로 확장합니다. 크레파스는 특별 보상으로, 스케치북에 적용하면 흑백 선화였던 세계에 색이 생깁니다. 마지막에는 숲 끝의 문을 열고 현실로 돌아갑니다.

# 핵심 차별점

## 1. 스케치북이 단순 UI가 아니라 게임 진행의 중심

스케치북은 미니맵이나 장식이 아니다. 플레이어는 스케치북에서 직접 세계를 그리고 색칠한다. 그 결과가 실제 필드에 반영된다.

## 2. 연필 조각을 경험치처럼 사용하지만 테마와 결합

일반적인 경험치 바 대신 연필 조각이 모여 하나의 연필이 된다. 이 연필로 새 구역을 그리기 때문에 자원과 세계관이 자연스럽게 연결된다.

## 3. 크레파스로 세계에 색을 되찾는 보상 구조

크레파스는 단순 수집물이 아니라 필드 전체의 색상 레이어를 바꾸는 특별 보상이다. 플레이어가 탐험할수록 세계가 점점 따뜻하게 변한다.

## 4. 텍스트 없는 직관적 진행

NPC 대사와 퀘스트 설명을 문장으로 쓰지 않고, 아이콘 말풍선과 상호작용 힌트로 전달한다. 브라우저 게임답게 빠르게 이해하고 플레이할 수 있다.

## 5. 10분 안에 완결되는 감성 경험

긴 성장 게임이 아니라, 짧은 시간 안에 시작, 탐험, 확장, 색칠, 엔딩을 모두 경험할 수 있는 MVP 구조다.

# 시연 순서

권장 시연 흐름은 다음과 같다.

```text
1. 도입 연출
2. 오두막 내부 조작 시작
3. 오두막 밖으로 나가 스케치북 세계 전환
4. NPC 아이콘 퀘스트 확인
5. 퀘스트 오브젝트와 연필 조각 획득
6. 연필 완성 UI 확인
7. 오두막 위 느낌표와 복귀 유도
8. 스케치북에서 숲 영역 확장
9. 크레파스 획득과 색 적용
10. 3차 확장 후 문 등장
11. 문을 열고 엔딩
12. 완성된 스케치북 화면 확인
```

# 시연 중 강조할 장면

## 도입 연출

강조 문장:

```text
처음부터 세계를 설명하는 문장 대신,
아이가 직접 스케치북에 오두막과 마을을 그리는 장면으로 설정을 전달했습니다.
```

## 연필 완성

강조 문장:

```text
경험치를 단순 수치로 보이지 않고,
연필 조각이 모여 세계를 더 그릴 수 있는 연필이 되는 방식으로 표현했습니다.
```

## 스케치북 확장

강조 문장:

```text
플레이어가 새 지역을 해금하는 것이 아니라,
스케치북에 직접 숲을 그려서 실제 필드를 확장합니다.
```

## 크레파스 색 적용

강조 문장:

```text
크레파스는 특별 보상입니다.
획득 후 스케치북에 적용하면 전체 세계에 색이 생겨 탐험의 결과가 시각적으로 남습니다.
```

## 엔딩

강조 문장:

```text
마지막 문은 시작점이었던 오두막 문과 닮아 있습니다.
아이는 문을 열고 현실로 돌아오지만,
스케치북에는 지금까지 완성한 세계가 그대로 남습니다.
```

# AI 코딩 활용 포인트

이 프로젝트는 AI 코딩 도구와 잘 맞는다.

활용 가능한 영역:

- Vite + TypeScript 프로젝트 생성

- 연필 UI와 크레파스 UI

사람이 집중해야 하는 영역:

- 게임 규칙 결정
- 무문자 UX 설계
- 진행 속도 조정
- 최종 연출 선택
- 플레이 감정 조율

# 예상 질문과 답변

## 왜 텍스트를 거의 쓰지 않았나요?

게임의 핵심이 스케치북과 그림이기 때문에, 설명도 최대한 그림과 아이콘으로 전달하려고 했습니다. 플레이어가 글을 읽는 것보다, 오브젝트의 반짝임과 말풍선 아이콘을 보고 자연스럽게 행동하게 만드는 것이 목표입니다.

## 왜 전투가 없나요?

Pencil Village는 힐링 탐험 게임입니다. 실패나 긴장보다 발견, 확장, 완성을 핵심 감정으로 잡았기 때문에 체력, 제한 시간, 전투는 MVP에서 제외했습니다.

## 연필 조각은 어떤 역할인가요?

연필 조각은 경험치 역할을 하지만, 일반적인 수치가 아니라 세계를 더 그릴 수 있는 연필을 완성하는 재료입니다. 연필이 완성되면 스케치북에 새 숲 영역을 그릴 수 있습니다.

## 크레파스는 필수인가요?

크레파스는 특별 보상입니다. 색을 적용하면 필드가 더 풍부해지고, 엔딩 스케치북에도 적용한 색이 반영됩니다. MVP에서는 플레이 경험을 강화하는 보상으로 사용합니다.

## 가장 중요한 구현 포인트는 무엇인가요?

스케치북 UI와 실제 필드 상태가 연결되는 구조입니다. 스케치북에서 새 숲을 그리면 외부 필드의 이동 가능 영역이 확장되어야 합니다.

```text
Pencil Village는 연필로 그린 세계를 직접 완성해 나가는 힐링 탐험 게임입니다.
플레이어는 연필 조각으로 세계의 선을 확장하고,
크레파스로 세계에 색을 입히며,
마지막에는 자신이 그린 문을 열고 현실로 돌아갑니다.
```

# 데모 체크리스트

시연 전에 다음을 확인한다.

- 게임이 브라우저에서 바로 실행된다.
- 시작 연출이 정상적으로 나온다.
- 조작 키가 동작한다.
- 상호작용 키캡이 보인다.
- 첫 퀘스트를 완료할 수 있다.
- 연필 조각 3개로 연필이 완성된다.
- 스케치북에서 숲이 확장된다.
- 크레파스 색이 적용된다.
- 문이 등장한다.
- 엔딩이 정상적으로 나온다.

# 향후 확장 가능성

- 마을 건물 추가
- 주민 증가
- 추가 크레파스 색상
- 숨겨진 상호작용 오브젝트
- 더 많은 숲 구역
- 엔딩 스케치북에 NPC 등장
- 저장 기능
- 모바일 터치 대응

# 최종 메시지

Pencil Village의 핵심은 많은 기능이 아니라, **플레이어가 세계를 직접 완성했다는 감각**이다. 발표에서는 기술 설명보다 이 감각을 시연 장면으로 보여주는 것이 가장 중요하다.
