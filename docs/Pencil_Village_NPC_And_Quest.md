# 문서 목적

이 문서는 Pencil Village의 NPC 구성, 아이콘 기반 퀘스트, 보상 흐름, 퀘스트 상태 관리 규칙을 정의한다.

# NPC 목록

MVP 기준 NPC는 4종이다.

- 우체통 주민
- 작은 새
- 다람쥐
- 큰 나무

각 NPC는 텍스트 대사가 아니라 아이콘 말풍선으로 의사소통한다.

# 퀘스트 구조

확장 단계마다 필수 NPC 퀘스트 1개가 있다. 퀘스트 보상은 연필 조각 1개다. 나머지 연필 조각 2개는 필드 상호작용으로 얻는다.

```text
확장 1회에 필요한 연필 조각: 3개
NPC 퀘스트 보상: 1개
필드 수집: 2개
```

크레파스는 큰 나무 또는 특정 보상 상호작용을 통해 얻는 특별 보상이다.

# 아이콘 말풍선 문법

## 요청 상태

```text
[필요한 물건] + [?]
```

## 완료 가능 상태

```text
[필요한 물건] + [하트]
```

## 보상 상태

```text
[연필 조각] + [반짝임]
```

또는

```text
[크레파스] + [반짝임]
```

# 공통 퀘스트 흐름

```text
1. NPC가 반짝임으로 상호작용 가능함을 표시한다.
2. 플레이어가 가까이 가면 머리 위에 키캡 말풍선이 뜬다.
3. 상호작용하면 NPC 말풍선에 필요한 아이콘이 나온다.
4. 플레이어가 필드에서 해당 오브젝트를 찾는다.
5. 오브젝트와 상호작용해 획득한다.
6. NPC에게 돌아간다.
7. NPC와 다시 상호작용한다.
8. NPC가 완료 아이콘을 표시한다.
9. 보상 오브젝트가 필드에 떨어진다.
10. 플레이어가 보상과 상호작용해 획득한다.
```

자동 획득은 없다.

# NPC 1: 우체통 주민

## 역할

첫 번째 필수 퀘스트를 제공한다. 플레이어가 NPC 퀘스트 문법을 처음 배우는 대상이다.

## 위치

초기 마을 영역의 우체통 근처.

## 요청 아이콘

```text
[편지] + [?]
```

## 플레이어 행동

마을 근처에 떨어진 편지를 찾는다. 편지는 반짝이는 오브젝트로 표시한다.

## 보상

- 연필 조각 1개

## 디자인 의도

편지는 형태가 명확하고 마을 분위기와 잘 맞는다. 텍스트 없이도 우체통 주민이 편지를 찾고 있다는 것을 이해하기 쉽다.

# NPC 2: 작은 새

## 역할

숲 영역 1의 필수 퀘스트를 제공한다.

## 위치

숲 영역 1의 작은 나무 또는 둥지 근처.

## 요청 아이콘

```text
[나뭇가지] + [?]
```

## 플레이어 행동

숲길 안에서 나뭇가지를 찾는다.

## 보상

- 연필 조각 1개

## 디자인 의도

작은 새와 나뭇가지는 자연스럽게 연결된다. 텍스트 없이도 둥지를 만들거나 고치기 위해 나뭇가지가 필요하다고 해석할 수 있다.

# NPC 3: 다람쥐

## 역할

숲 영역 2의 필수 퀘스트를 제공한다.

## 위치

숲 속 공터 또는 큰 나무 뿌리 근처.

## 요청 아이콘

```text
[도토리] + [?]
```

## 플레이어 행동

공터 근처의 도토리를 찾는다.

## 보상

- 연필 조각 1개

## 디자인 의도

다람쥐와 도토리는 매우 직관적인 관계다. 아이콘 기반 퀘스트에 적합하다.

# NPC 4: 큰 나무

## 역할

큰 나무는 크레파스 보상 퀘스트와 마지막 상징적 안내 역할을 맡는다.

## 위치

숲 영역 3 중심부 또는 문으로 가는 길목.

## 요청 아이콘

```text
[빛나는 잎] + [?]
```

## 플레이어 행동

숲 영역 3 안에서 빛나는 잎을 찾는다.

## 보상

- 크레파스

## 디자인 의도

큰 나무는 세계에 색을 되찾게 해주는 상징적인 존재다. 빛나는 잎을 가져오면 크레파스를 떨어뜨리는 방식으로 보상을 준다.

# 크레파스 보상 방식

크레파스는 퀘스트 완료 즉시 자동 획득되지 않는다.

흐름:

```text
큰 나무 퀘스트 완료
-> 큰 나무가 반짝임 또는 하트 아이콘 표시
-> 크레파스가 바닥에 떨어짐
-> 크레파스가 반짝임
-> 플레이어가 가까이 접근
-> 키캡 말풍선 표시
-> 상호작용해 획득
```

이후 플레이어가 오두막 안의 스케치북과 상호작용하면 색이 적용된다.

# 크레파스 색상 가이드

기획상 유도 순서는 다음과 같다.

```text
초록색 -> 갈색 -> 하늘색
```

하지만 실제 적용은 플레이어가 획득한 순서에 따른다.

색상별 효과:

- 초록색: 풀, 나뭇잎, 덤불
- 갈색: 나무줄기, 오두막, 울타리, 흙길
- 하늘색: 물, 하늘빛 효과, 문 주변 빛

# 퀘스트 상태

각 퀘스트는 다음 상태를 가진다.

```text
locked: 아직 등장하지 않음
available: NPC가 요청 가능
accepted: 요청을 본 상태
itemCollected: 필요한 물건을 얻음
completed: NPC에게 전달 완료
rewardDropped: 보상이 필드에 떨어짐
rewardCollected: 보상 획득 완료
```

MVP에서는 `accepted`를 별도로 복잡하게 쓰지 않아도 된다. NPC 말풍선을 한 번 본 뒤 필요한 아이템을 찾아오면 된다.

# 퀘스트 배치 원칙

텍스트가 없으므로 퀘스트 오브젝트는 너무 멀리 있으면 안 된다.

권장:

- NPC가 있는 구역 안에 배치한다.
- 반짝임으로 상호작용 가능함을 알린다.
- 오브젝트 형태가 요청 아이콘과 동일해야 한다.
- 다른 비슷한 오브젝트와 헷갈리지 않게 한다.

# 필드 연필 조각

NPC 퀘스트 보상 외에도 각 단계마다 연필 조각 2개를 필드에 배치한다.

규칙:

- 반짝임으로 표시한다.
- 자동 획득하지 않는다.
- 상호작용 시 연필 UI에 조각이 추가된다.
- 단계별로 필요한 조각을 모두 얻으면 연필이 완성된다.

# 퀘스트 우선순위

MVP 구현 순서:

```text
1. 우체통 주민과 편지 퀘스트
2. 연필 조각 보상 드롭
3. 작은 새와 나뭇가지 퀘스트
4. 다람쥐와 도토리 퀘스트
5. 큰 나무와 빛나는 잎 퀘스트
6. 크레파스 보상 드롭
```

# 퀘스트 완성 기준

NPC and Quest 시스템이 완성되었다고 판단하려면 다음이 가능해야 한다.

- NPC에게 다가가면 상호작용 키캡이 뜬다.
- NPC 말풍선에 필요한 물건 아이콘이 표시된다.
- 해당 물건을 필드에서 찾고 상호작용해 얻을 수 있다.
- NPC에게 돌아가면 퀘스트가 완료된다.
- 보상이 필드에 떨어진다.
- 보상과 상호작용해야 실제로 획득된다.
- 퀘스트 보상 연필 조각이 연필 UI에 반영된다.
