# 문서 목적

이 문서는 Pencil Village의 맵 구조, 영역 확장, 카메라, 오두막 내부, 외부 필드, 최종 문 위치를 정의한다.

# 기본 맵 구조

게임은 크게 두 종류의 공간으로 구성된다.

```text
오두막 내부: 별도 맵
마을과 숲: 하나의 큰 외부 필드
```

오두막 내부는 현실과 스케치북 세계의 경계 역할을 한다. 외부 필드는 스케치북에 그려진 마을과 숲이 실제로 펼쳐진 공간이다.

# 화면 방식

- 탑다운 2D
- 약간 사선에서 보는 느낌
- 스타듀밸리처럼 위에서 내려다보되, 오브젝트의 앞면이 일부 보인다.
- 굶지마처럼 손그림 감성 허용

# 카메라

외부 필드는 하나의 큰 맵이므로 카메라는 플레이어를 따라간다.

기본 규칙:

- 카메라는 플레이어 중심으로 부드럽게 이동한다.
- 맵 끝에서는 카메라가 더 이상 이동하지 않는다.
- 연필 완성 후 오두막 근처 접근 시 카메라 또는 캐릭터 시선이 오두막 쪽을 살짝 유도할 수 있다.

# 오두막 내부

오두막 내부는 별도 씬 또는 별도 맵으로 처리한다.

## 주요 오브젝트

- 책상
- 스케치북
- 침대
- 러그
- 모닥불
- 오두막 문

## 역할

오두막은 다음 기능을 수행한다.

- 게임 시작 지점
- 스케치북 확인 장소
- 숲 확장 장소
- 크레파스 색 적용 장소
- 엔딩 직전 진행 확인 장소

## 스케치북 위치

스케치북은 책상 위에 놓여 있다. 항상 상호작용 가능하다.

상호작용 결과:

- 연필이 있으면 다음 숲 영역 그리기
- 크레파스가 있으면 색상 적용
- 아무것도 없으면 현재 스케치북 지도 표시

# 외부 필드

외부 필드는 마을과 숲이 연결된 하나의 큰 공간이다. 구역 전환 화면을 쓰지 않고 플레이어가 자연스럽게 이동할 수 있다.

## 초기 상태

처음 바깥으로 나가면 짧은 연출로 마을과 숲의 선이 그려진다.

초기 이동 가능 영역:

- 오두막 앞
- 작은 중세풍 마을
- 초기 숲 입구

오른쪽 방향은 아직 비어 있거나 흐릿한 밑그림으로 막혀 있다.

# 진행 단계

전체 진행은 다음 구조다.

```text
초기 영역
-> 숲 영역 1
-> 숲 영역 2
-> 숲 영역 3
-> 최종 문
```

# 초기 영역

## 구성

- 오두막 외부
- 작은 마을 광장
- 우체통 주민
- 초기 숲 입구
- 첫 번째 필수 퀘스트 오브젝트
- 연필 조각 2개

## 목표

- 우체통 주민 퀘스트 완료
- 필드 연필 조각 2개 획득
- 연필 조각 3개로 연필 완성
- 오두막으로 돌아가 숲 영역 1 확장

# 숲 영역 1

## 해금 조건

- 첫 번째 연필 완성
- 스케치북에서 1차 확장 완료

## 구성

- 더 깊어진 숲길
- 작은 새 NPC
- 나뭇가지 퀘스트 오브젝트
- 연필 조각 2개
- 크레파스 보상 가능 지점

## 목표

- 작은 새 퀘스트 완료
- 연필 조각 3개 수집
- 오두막으로 돌아가 숲 영역 2 확장

# 숲 영역 2

## 해금 조건

- 두 번째 연필 완성
- 스케치북에서 2차 확장 완료

## 구성

- 숲 속 공터
- 다람쥐 NPC
- 도토리 퀘스트 오브젝트
- 연필 조각 2개
- 크레파스 보상 가능 지점

## 목표

- 다람쥐 퀘스트 완료
- 연필 조각 3개 수집
- 오두막으로 돌아가 숲 영역 3 확장

# 숲 영역 3

## 해금 조건

- 세 번째 연필 완성
- 스케치북에서 3차 확장 완료

## 구성

- 문으로 이어지는 마지막 숲길
- 큰 나무 NPC
- 빛나는 잎 퀘스트 오브젝트
- 크레파스 보상
- 최종 문 등장 위치

## 목표

- 문 완성 연출 확인
- 필드에서 문까지 이동
- 문과 상호작용해 엔딩 진입

# 최종 문 위치

문은 숲 영역 3 끝에 배치한다. 외형은 오두막 문과 닮아 있어야 한다.

의도:

- 게임 시작 때 오두막 문으로 밖에 나간다.
- 게임 끝에서는 오두막 문과 닮은 문으로 현실에 돌아간다.
- 시작과 끝이 문으로 연결된다.

# 스케치북 지도 UI

스케치북 UI는 실제 필드의 완전한 복사본이 아니라, 큰 지도처럼 축약된 진행도 표시다.

## 기본 표시

```text
[오두막] - [마을] - [숲] - [숲1] - [숲2] - [숲3] - [문]
```

처음에는 오른쪽 영역 대부분이 비어 있다. 문은 첫 스케치북 확인 시 희미하게 보인다.

## 상태별 표시

- 아직 그려지지 않은 구역: 흐릿한 밑그림 또는 빈 공간
- 해금된 구역: 진한 연필 선
- 색이 적용된 요소: 획득한 크레파스 색상 반영
- 최종 문: 3차 확장 후 선명한 열린 문으로 표시

# 이동 제한

아직 그려지지 않은 영역은 이동할 수 없다.

표현 방식:

- 길이 끊겨 있음
- 흐릿한 밑그림으로만 보임
- 종이의 빈 공간처럼 보임
- 플레이어가 지나가려 하면 더 이상 이동하지 않음

텍스트로 “잠김”을 표시하지 않는다.

# 맵 배치 가이드

## 플레이어 동선

플레이어 동선은 왼쪽에서 오른쪽으로 확장되는 느낌을 유지한다.

```text
오두막 / 마을 -> 숲 입구 -> 숲1 -> 숲2 -> 숲3 -> 문
```

## 오브젝트 배치

퀘스트 오브젝트는 해당 NPC와 너무 멀리 두지 않는다. 텍스트 없는 게임이므로 플레이어가 무엇을 찾아야 하는지 시각적으로 연결할 수 있어야 한다.

권장 거리:

- 초기 영역: NPC에서 화면 1-2개 거리
- 숲 영역 1: NPC 주변 숲길 안
- 숲 영역 2: 공터 안 또는 바로 옆
- 큰 나무: 숲 영역 3 중심부

# 맵 구현 우선순위

MVP에서 우선 구현할 것:

- 오두막 내부
- 외부 필드 기본 충돌
- 초기 마을과 숲
- 숲 영역 1-3의 해금 상태
- 카메라 추적
- 최종 문 위치

나중에 추가할 수 있는 것:

- 장식용 마을 건물 변화
- 더 많은 숲 오브젝트
- 숨어 있는 상호작용 요소
- 엔딩에서 NPC 작은 그림

# 진행 완성 기준

Map and Progression이 완성되었다고 판단하려면 다음이 가능해야 한다.

- 오두막 내부와 외부 필드를 오갈 수 있다.
- 외부 필드에서 플레이어 중심 카메라가 작동한다.
- 초기 영역만 먼저 이동 가능하다.
- 스케치북 확장 후 숲 영역 1-3이 순차적으로 이동 가능해진다.
- 3차 확장 후 문이 숲 끝에 등장한다.
