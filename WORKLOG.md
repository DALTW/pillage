# Unity Work Branch (5/18~5/20)

## 5/18 작업 내용
- 오두막 내부 맵 구성
- 플레이어 이동 구현
- 카메라 추적 구현
- 스케치북 상호작용 구현
- 문 상호작용 구현
- 스케치북/문 E 말풍선 표시 구현
- 소년 캐릭터 및 대기/달리기 애니메이션 리소스 추가

## 5/18 확인 방법
- Unity에서 SampleScene 실행
- 방향키 또는 WASD로 플레이어 이동 확인
- 플레이어 이동 시 카메라 추적 확인
- 스케치북 근처로 이동하면 E 말풍선 표시 확인
- E 키로 스케치북 UI 열기 확인
- 스케치북 확인 후 문 상호작용 확인

## 5/19 작업 내용
- BigSketchbook UI 열림/닫힘 흐름 정리
- 스케치북이 처음 열릴 때 연필이 지나가며 그림이 그려지는 애니메이션 구현
- 스케치북 안에 선으로 된 집과 숲 앞 입구 그림 추가
- 그림 애니메이션이 첫 스케치북 확인 시에만 실행되도록 구현
- 이후 스케치북을 다시 열면 완성된 그림만 바로 표시되도록 구현
- 우체통이 있는 두 번째 집 선화 추가
- 스케치북 그림 렌더링용 UI 선화/연필 그래픽 스크립트 추가

## 5/19 확인 방법
- Unity에서 SampleScene 실행
- 스케치북 근처로 이동 후 E 키 입력
- BigSketchbook UI가 열린 뒤 연필이 지나가며 그림이 그려지는지 확인
- 선으로 된 집, 숲 앞 입구, 우체통이 있는 두 번째 집 표시 확인
- 스케치북을 닫았다가 다시 열면 연필 애니메이션 없이 완성된 그림만 표시되는지 확인
- 스케치북 확인 후 문 상호작용이 정상 동작하는지 확인

## 5/20 작업 내용
- 스케치북 드로잉 시간을 18초로 늘리고 연필 크기/길이 감소/흔들림/멈칫거림을 추가해 아이가 그리는 듯한 연출로 보정
- 첫 스케치북 드로잉 중 E, Escape, 트리거 이탈, 플레이어 이동으로 연출을 건너뛰지 못하도록 잠금 처리
- 플레이 시작 시 스케치북 확인 상태가 초기화되고 첫 드로잉 완료 후에만 진행 상태가 저장되도록 보강
- 내부 문은 스케치북 1회 확인 후에만 열리도록 강제하고 outside 이동/복귀 전환 흐름 구현
- outside 진입 시 흰 배경이 펼쳐지고 선화 집이 그려지며, 한 번 완성된 outside 맵은 실내/외부 이동 후에도 완성 상태 유지
- outside 맵 크기를 40x28로 정리하고 울타리 선화와 BoxCollider2D 경계 벽을 추가
- 사용자 grass/tree/mailbox 이미지를 Resources/Outside 소품으로 추가하고 굵은 선/투명 배경/스프라이트 메타데이터 보정
- outside에 이웃집+mailbox, grass, tree를 배치하고 소품 크기, 바닥선, sorting order를 조정
- grass/tree 랜덤 배치를 고정 seed로 생성하고 제외 구역, 스폰 간격, 화면상 영역 겹침 방지를 적용
- 오른쪽 아래 숲 입구의 tree 3개를 제거하고 선그림 숲 입구 오브젝트로 대체
- 스케치북 안의 나무 선화를 outside tree 형태로 교체
- 상호작용 대상 탐색을 오브젝트 중심이 아니라 콜라이더 기준 거리로 보강

## 5/20 확인 방법
- Unity에서 SampleScene 실행
- 스케치북을 처음 열었을 때 연필이 아이가 그리는 듯한 흔들림/멈칫거림으로 그림을 그리는지 확인
- 스케치북 드로잉 중 E, Escape, 범위 이탈, 이동으로 연출이 건너뛰어지지 않는지 확인
- 스케치북 드로잉 완료 후 내부 문으로 outside 이동이 되는지 확인
- outside 진입 시 흰 배경과 선화 집이 나타나고 다시 들어와도 완성 상태가 유지되는지 확인
- outside 맵의 울타리 경계 밖으로 플레이어가 나가지 않는지 확인
- grass/tree/mailbox가 투명 배경으로 보이고 tree/grass가 서로 겹치지 않게 배치되는지 확인
- 오른쪽 아래 숲 입구가 tree 3개가 아닌 선그림 숲 입구 오브젝트로 보이는지 확인
- 스케치북 안의 나무가 outside tree 형태와 비슷하게 보이는지 확인
