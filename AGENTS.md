# AGENTS.md

이 파일은 `Pencil Village` 프로젝트에서 AI 코딩 에이전트와 개발자가 따라야 할 작업 지침이다. 프로젝트의 세부 기획은 `docs/` 문서를 우선 기준으로 삼는다.

## 프로젝트 요약

- 제목: **Pencil Village**
- 목표: 10분 안에 처음부터 엔딩까지 플레이 가능한 브라우저 힐링 탐험 게임 MVP
- 스택: Vite + TypeScript + Canvas API
- 플랫폼: 브라우저 싱글플레이
- 핵심 루프: 탐험 -> NPC 퀘스트 -> 연필 조각 획득 -> 연필 완성 -> 스케치북 확장 -> 새 구역 탐험 -> 최종 문
- 핵심 UX: 일반 대사와 튜토리얼 문장을 최소화하고, 아이콘 말풍선과 상호작용 힌트로 진행을 전달한다.

## 우선 읽을 문서

작업 전 목적에 맞는 문서를 먼저 확인한다.

- 전체 인덱스: `docs/README_Project_Document_Index.md`
- 게임 정체성: `docs/Pencil_Village_Game_Concept.md`
- 핵심 루프: `docs/Pencil_Village_Core_Gameplay.md`
- 구현 명세: `docs/Pencil_Village_Technical_Spec.md`
- MVP 범위: `docs/Pencil_Village_MVP_Scope.md`
- 진행 관리: `docs/TODO.md`

개발에 바로 들어갈 때는 다음 순서를 우선한다.

```text
1. Technical Spec
2. Core Gameplay
3. Map and Progression
4. Interaction UX
5. MVP Scope
6. TODO
```

## 작업 원칙

### 1. 생각하고 구현하기

- 불명확한 요구사항은 추측하지 말고 드러낸다.
- 해석이 여러 개인 경우 선택지를 설명한다.
- 더 단순한 접근이 있으면 먼저 제안한다.
- 구현 전 성공 기준을 확인한다.

### 2. 브랜치와 커밋 관리

- 새로운 기능 개발 시 `main`에서 직접 작업하지 않고 기능 브랜치를 분리한다.
- 기능 브랜치 이름은 `codex/` prefix를 붙인다.
- 기능이 완성되고 검증되면 `main` branch로 병합한다.
- 기능 개발 시 하위 목표를 생성하고, 각 하위 목표가 달성될 때마다 commit을 작성한다.
- commit은 완료된 하위 목표와 검증 내용을 추적할 수 있게 작게 유지한다.

### 3. 단순하게 유지하기

- 요청받지 않은 기능을 추가하지 않는다.
- 한 번만 쓰는 코드를 위한 추상화를 만들지 않는다.
- MVP에 없는 저장, 서버, 로그인, 전투, 체력, 제한 시간, 랜덤 생성은 추가하지 않는다.
- 10분 안에 완주 가능한 경험을 우선한다.

### 4. 작게 변경하기

- 변경 범위를 요청과 직접 관련된 파일로 제한한다.
- 기존 문서와 코드 스타일을 따른다.
- 관련 없는 리팩터링, 정리, 포맷 변경을 섞지 않는다.
- 변경으로 생긴 미사용 코드나 문서 항목은 함께 정리하되, 기존의 무관한 항목은 건드리지 않는다.

### 5. 검증 중심으로 진행하기

- 기능 작업은 재현 가능한 완료 기준을 먼저 둔다.
- 구현 후 가능한 검증을 실행한다.
- 프론트엔드 변경 후에는 로컬 실행과 브라우저 확인을 우선한다.
- 검증하지 못한 항목은 최종 응답에 명시한다.

## 구현 지침

- Canvas 기반 게임 루프, 씬 전환, 입력, 카메라, 상태 관리를 먼저 세운다.
- 스케치북 UI와 실제 필드 상태가 반드시 연결되어야 한다.
- 모든 핵심 획득은 자동 획득이 아니라 상호작용으로 처리한다.
- 상호작용 대상은 반짝임, 플레이어 머리 위 말풍선, 키캡 아이콘 규칙을 일관되게 따른다.
- 연필 조각 3개로 연필 1개가 완성되고, 연필 1개로 숲 영역 1개가 확장된다.
- 크레파스는 획득 즉시 적용하지 않고, 오두막의 스케치북에서 적용한다.
- 연필과 크레파스를 동시에 보유하면 연필 적용을 우선한다.
- 일반 NPC 대사와 튜토리얼 문장은 MVP에서 사용하지 않는다.

## 문서 관리 지침

- `README.md`는 프로젝트 소개와 실행, 문서 안내의 입구로 유지한다.
- `docs/TODO.md`는 전체 목표, 현재 진행, 완료 여부를 관리하는 작업 보드로 사용한다.
- 모든 할 일과 완료한 일은 `docs/TODO.md`에서 관리한다.
- 완료된 항목은 체크박스를 갱신하고, 완료 기준 또는 확인 방법을 함께 남긴다.
- 중요한 범위 변경은 `docs/TODO.md`의 결정 기록에 추가한다.
- 기획 변경이 생기면 관련 `docs/Pencil_Village_*.md` 문서와 TODO가 서로 모순되지 않게 갱신한다.

## 완료 기준

MVP 완료는 최소한 다음 조건을 만족해야 한다.

- 브라우저에서 새로고침 후 처음부터 플레이 가능하다.
- 도입 연출 후 오두막 내부에서 조작 가능하다.
- 오두막 내부와 외부 필드를 오갈 수 있다.
- NPC 아이콘 퀘스트와 필드 수집으로 연필 조각을 얻을 수 있다.
- 스케치북에서 숲 영역을 3회 확장할 수 있다.
- 크레파스를 스케치북에서 적용할 수 있다.
- 3차 확장 후 최종 문이 등장한다.
- 문을 열면 엔딩과 완성된 스케치북 화면이 나온다.
- 일반 대사 없이도 기본 진행이 가능하다.
